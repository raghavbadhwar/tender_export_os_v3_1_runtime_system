# Tender Export OS 90+ Upgrade Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Raise Tender Export OS from ~78/100 to 90+ by turning the current strong local operating system into a field-operational, mobile-delivered, Drive-synced, source-tested, plugin-exercised business OS.

**Architecture:** Keep Hermes as control plane, Kanban/cron/approvals as operating rhythm, Codex as artifact/runtime factory, ChatGPT as boardroom research layer, and Google Drive as source-of-truth projection. Add operational integrations in approval-safe layers: mobile delivery, Drive sync, live source adapters, plugin execution receipts, dashboards, and end-to-end case simulations.

**Tech Stack:** Hermes Agent, Hermes cron, Hermes Kanban, Python scripts, CSV/JSON/YAML registers, Google Workspace `gws` CLI, local Claude plugins, Accio skill library, browser automation/Playwright where approved, Telegram/WhatsApp/Slack gateway if owner approves.

## Risks and Guardrails

| Risk | Guardrail |
|---|---|
| Credentials leak during gateway/Drive setup | Never print credential file contents; redact secrets; store only config references |
| Plugins send messages unintentionally | Keep messaging plugins disabled until owner approval; draft-only mode first |
| Source scraping violates terms or hits CAPTCHA | Use public allowed sources only; no CAPTCHA bypass; browser screenshots only where permitted |
| Automation bypasses approval gates | Mobile approval creates receipts; execution remains separate and explicit |
| Drive becomes inconsistent with local records | Use `data/drive_manifest.csv` with checksums and receipts |
| Dashboards become stale | Generate during morning/evening cron and include timestamp/source files |
| LLM regression causes unsafe claims | Add prompt audit and structural regression checks |
