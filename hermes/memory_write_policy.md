# Hermes Memory Write Policy

## Default
Memory writes are enabled but approval-gated.

Config source: `config/memory_policy.yaml`

## Save
- durable founder preferences
- repeated corrections
- future workflow rules
- source quirks
- supplier lessons summarized compactly
- category preferences
- important tool/runtime quirks
- approval preferences

## Do Not Save
- raw tenders
- raw RFQs
- long supplier tables
- full PDFs
- passwords
- credentials
- DSC files
- bank details
- sensitive contract details
- temporary paths
- one-off facts
- unverified supplier claims as facts

## Procedure
1. Propose a compact memory update.
2. Explain why it will help future runs.
3. Ask for owner approval.
4. Write only after approval.
5. Log the memory update proposal/action.
